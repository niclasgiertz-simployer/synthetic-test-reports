6 Issues

  • 🔴
    Sick leave not reflected in Time tab Critical
    After reporting sick via chat, the Time tab's "My Requests" section doesn't show it. Only vacation requests appear. This creates fundamental uncertainty about whether the action was actually registered in the system — the kind of doubt that makes an employee email HR to double-check.
  • 🔴
    Manager-oriented content in Employee view Critical
    The Inbox empty state says "Check in on your team or prep for upcoming 1:1s" (manager language). The Time tab shows a "Team Requests" approval queue. Quick Actions includes a "Manager" category. None of this should be visible to an employee-role user like Robert.
  • 🟡
    No follow-up buttons after extending sick leave Medium
    After the initial sick report, three contextual buttons appear (Extend, Cancel, Who covers meetings). But after clicking "Extend to tomorrow," no new buttons are offered — Robert has to type manually, which is more friction when you're unwell.
  • 🟡
    No sick leave policy information surfaced Medium
    When reporting sick, no proactive information about self-certification periods, doctor's note thresholds, or a link to the sick leave policy is shown. This is the exact moment where this information would be most useful and appreciated.
  • 🟡
    No option to report sick for a future date Medium
    "Report Sick" immediately registers for today with no option to specify a date or range. What about a planned medical procedure tomorrow? There's no date selection step in the flow.
  • 🟡
    Slow streaming response for transactional action Medium
    The AI response streams word-by-word over ~10 seconds. For a simple transactional confirmation, watching text slowly appear feels anxious. An instant confirmation card with supplementary AI text streaming below would feel faster and more reliable.
  • 🟢
    Text-based confirmation vs. visual card Low
    The confirmation is plain text in a chat bubble with markdown bold. A styled card with icon, date range, status badge, and "Manager notified" indicator would feel more official and scannable.
  • 🟢
    No confirmation email/notification option Low
    No way to get an email or push notification confirming the sick leave for personal records. A "Send me a confirmation" button would provide reassurance.

🔴 Sync sick leave to Time tab

In the Sia MVP prototype (sia-mvp.vercel.app), when a user reports sick via the chat assistant, the sick leave is NOT reflected in the Time tab's "My Requests" section. Only vacation requests appear there.

Fix this so that after Sia confirms a sick leave report, an entry immediately appears in the Time tab under "My Requests" with:
- Type: "Sick Leave" (with the 🤒 emoji icon)
- Date range: the dates reported (e.g., "Mar 4" or "Mar 4-5" if extended)
- Status: "Active" (with a distinct color, e.g., orange or the existing yellow style)

This entry should update in real-time if the user extends the sick leave via chat. The sick days counter at the top of the Time tab should also increment accordingly.

This is the highest-priority fix — without it, users lose trust in the system because there's no persistent record of their action outside the chat history.

🔴 Scope content to user role

In the Sia MVP prototype, the employee view currently shows manager-only content in multiple places. Fix the following for the Employee role:

1. TIME TAB — Hide the "Team Requests" section (the approval queue showing e.g., "Kalle Svensson Jul 15-21"). Employees don't approve requests. Keep the "Upcoming" team absence list as read-only visibility is fine.

2. INBOX — Change the empty state message from "No pending tasks. Check in on your team or prep for upcoming 1:1s." to "All caught up! No pending notifications." The current copy is clearly manager-oriented.

3. QUICK ACTIONS — Hide the "Manager" category (My Team, Team Well-being, Team Absence, Prepare 1:1) from the Quick Actions panel when the logged-in user has an Employee role. Only show "Pinned" and "Personal" categories.

The role should be determined by the logged-in user's account type. Maria (manager) should still see all sections.

🟡 Add follow-up buttons after extension

In the Sia MVP chat flow, after the user reports sick, three follow-up buttons appear: "Extend to tomorrow", "Cancel sick report", "Who covers my meetings?" — these are great.

But after clicking "Extend to tomorrow", the response only says "If you need to extend further, just let me know" with NO follow-up buttons. The user has to type manually.

Add follow-up buttons after every sick leave response, not just the initial one. After an extension, show:
- "Extend one more day"
- "Cancel sick leave"
- "View sick leave policy"

The pattern should be: every Sia response about sick leave ends with contextually relevant quick-action buttons. Never leave the user with just a text input after a transactional action.

🟡 Surface sick leave policy proactively

In the Sia MVP, when a user reports sick, no policy information is shown. This is a missed opportunity — the moment someone reports sick is exactly when they'd wonder about rules.

After the sick leave confirmation message, add a subtle, collapsible info section:

"ℹ️ Good to know: Self-certification covers the first 3 days. After that, a doctor's note is required. [View full sick leave policy →]"

Style it as a muted/secondary element below the main confirmation — not as prominent as the confirmation itself, but visible enough that the user notices it. Use a slightly different background or a thin top border to visually separate it from the main message.

The policy link can point to a placeholder URL for now (e.g., #sick-leave-policy).

🟡 Support future-date sick leave

In the Sia MVP, clicking "Report Sick" immediately registers sick leave for today with no option to choose a date. This doesn't cover planned medical absences (e.g., surgery tomorrow).

Update the Report Sick flow so that after clicking the button, Sia asks:

"Are you reporting sick for today, or do you need to set a specific date?"

Show two quick-action buttons:
- "Today" (executes immediately as it does now)
- "Choose dates" (shows a simple date picker or asks "Which dates?" and accepts natural language like "tomorrow" or "March 10-12")

Keep the current one-tap flow as the fast path — "Today" should feel just as quick as before. The date selection is an optional branch for users who need it.

🟡 Show instant confirmation card

In the Sia MVP, the sick leave confirmation streams in word-by-word over ~10 seconds. For a transactional action, this feels slow and creates anxiety ("did it work?").

Add an instant confirmation card component that appears immediately (no streaming) when a sick leave action completes. The card should show:
- A green checkmark icon
- "Sick leave registered" (bold)
- Date range (e.g., "Wednesday, March 4")
- "Manager notified" badge with the manager's name
- "Calendar blocked" badge

Below the card, the conversational AI text ("Feel better soon!" etc.) can still stream in as supplementary content. The key insight: the structured confirmation should be instant, while the friendly chat text can stream.

Use a card style similar to the leave request cards in the Time tab for visual consistency.

🟢 Add email confirmation option

In the Sia MVP sick leave flow, add a follow-up button "Email me a confirmation" that would send the user a brief confirmation email with:
- Subject: "Sick leave confirmed — [date range]"
- Body: date range, manager notified, calendar blocked status

For the prototype, this button can show a toast/snackbar saying "Confirmation sent to your email" without actually sending. The important thing is that the option exists in the UI as a follow-up action.
