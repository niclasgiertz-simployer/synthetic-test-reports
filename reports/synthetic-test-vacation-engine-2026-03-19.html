What Worked Well

"The policy creation wizard is genuinely impressive. Selecting Netherlands gives me pre-configured statutory defaults — 4x weekly working days, Jan-Dec leave year, July 1 expiry for statutory days — all correct under Dutch law. Sweden is equally well done: 25 days, April-March leave year, summer rights for 4 consecutive weeks in June-August, förskottssemester toggle. The 'Required by law' badges give me instant confidence that compliance-critical settings are locked down. The Sia AI proactively found Dutch employees when I created the Netherlands policy and offered to add them — that's the kind of intelligent workflow automation I want. The 5-step wizard (Name → Statutory → Rules → Scope → Review) is logical and the review page gives me a full summary before committing. The policy scope flexibility (Country, Office, Dept, Title, Team, Employee) is excellent for a multi-country setup."
🚨

Critical Issues

"The biggest gap is that there's no way to transfer an employee between countries. Willem de Vries is in the Netherlands and I need to move him to Sweden — but the Employment tab is completely read-only with no edit button, no transfer action, nothing. The only action available is 'Offboard Employee.' In real life, international transfers are a core HR workflow, and without this, I'd have to offboard and re-hire, losing all history. Albania is a country I can select when adding an employee, but there's no pre-configured policy template for it and no obvious 'Custom policy' wizard was started from the country selection page — only a brief mention of 'Custom' at the bottom I couldn't fully explore. This means employees in unsupported countries get stuck with 'No policy' and no clear path forward."
💡

Key Opportunities

"Adding an employee transfer flow would be transformative — the system already knows about policies per country, so it could suggest 'Willem is moving to Sweden, would you like to switch from Netherlands Standard to Sweden Standard policy?' The Custom policy builder needs to be more prominent for unsupported countries. The 'Need help?' button and 'Connect with an expert' link are nice touches but I'd want to know what happens when I click them. Adding change history/audit trail to policies would build trust. The review page should fix the 'days days' bug and '1 countries' grammar issue."

🔴 Add employee transfer workflow

In the employee profile page (Employment tab), there is currently only a read-only display of employment details and an "Offboard Employee" button at the bottom. Add a "Transfer Employee" button next to or above the Offboard button.

When clicked, it should open a modal/wizard with:
1. New Country (dropdown, same country list as Add Employee)
2. New Office (dropdown, filtered by country)
3. Transfer Date (date picker)
4. A section showing current vacation policy and suggesting the matching policy for the new country (e.g., "Current: Netherlands Standard → Suggested: Sweden Standard")
5. Option to prorate remaining balance or carry over
6. A review step showing all changes
7. On confirmation: update employee's country, office, work location; unassign old policy; assign new country policy; add entry to Employment Timeline

The Employment tab fields (Job Title, Department, Office, Team, Work Location, etc.) should also become editable via an "Edit" icon/button in the section header.

🔴 Add 'Create Custom Policy' from assignment flow

In the Add Employee modal Step 2 (Vacation Policy Setup), when the "Select Policy" dropdown has no matching policy for the employee's country, add a "Create New Policy" button or link below the dropdown.

Similarly, in the "Assign Vacation Policy" modal on an employee's Vacation tab, when no policies match, instead of only showing "No policies configured" with "+ Create New Policy" that navigates away, add inline context: the new policy creation should know which employee triggered it. After policy creation, automatically redirect back to the employee profile with the new policy pre-selected for assignment.

Also, in the country selection page for Create Vacation Policy, make the "Custom" section more prominent — either as a card alongside the country cards or as a clear CTA for countries not in the pre-configured list.

🟡 Fix text bugs on review page

On the policy creation Step 5 (Review), fix two text display bugs:
1. The Entitlement field shows "4× weekly working days days" — the word "days" is duplicated. Find where the entitlement display string is constructed and remove the duplicate suffix.
2. The Selections field shows "1 countries, 3 employees" — should be "1 country, 3 employees". Add proper singular/plural handling: if count === 1, use singular form ("country", "employee"), otherwise use plural.

🟡 Add audit trail for policies

Add a "History" section to each vacation policy's detail/edit page. This should log:
- Policy created (timestamp, user)
- Any field changes (timestamp, user, field name, old value → new value)
- Employee added/removed from scope (timestamp, user, employee name)
- Policy deleted (timestamp, user)

Display as a timeline similar to the Employment Timeline component already used on the employee profile. Store events in the policy data model with: timestamp, userId, action type, field, oldValue, newValue.

🟡 Show policy name on employee Vacation tab

On the employee Vacation tab, above the "Vacation Balance" card, add a policy info banner showing:
- "Assigned Policy: [Policy Name]" (e.g., "Netherlands Standard Vacation Policy")
- Country flag + country name
- A link/button to view the full policy details
- An "Unassign" or "Change Policy" action

If no policy is assigned, show the current "No Vacation Policy Assigned" empty state (already exists).

🟡 Add success toast after employee creation

After successfully completing the Add Employee flow (clicking "Add Employee" on Step 2), show a toast notification:
- Message: "[Employee Name] has been added successfully"
- Include a "View Profile" link in the toast
- Auto-dismiss after 5 seconds
- Green success styling consistent with the app's design

🟢 Show calculated entitlement alongside formula

On the policy creation Step 2 (Statutory Configuration) and Step 5 (Review), when the Statutory Entitlement uses a formula like "4× weekly working days", add a helper text showing the calculated result based on the default work schedule.

For example: "4× weekly working days" → show "(= 20 days for 5-day week)" below or beside the field.

This helps HR Admins quickly verify the number without mental math, especially when comparing across countries.
